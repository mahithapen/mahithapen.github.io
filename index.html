Mit: Recreating Git on Terminal

This project revolves around building a system that mirrors and improves upon the functions in the version control system Git. This involves basic tasks such as reading and writing files, representing projects and directories as trees, abstracting files into ‚ÄúBlobs,‚Äù and comparing these elements for effective version control through the command line.

Through this project, I specifically worked on the functionality for users to access past repos and understand the changes made. I did this through the use of a linked list like structure that would store each commits parent. I then created a function to parse through a directory structure of trees and blobs (directories and files) to make comparisons.

To make the command for creating commits, the user passes a simple mit commit in the command line. The information of the commit are stored in the commit data structure.
